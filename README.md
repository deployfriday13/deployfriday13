# deployfriday

Python developer. I build CLI utilities, backend services, and telegram bots — tools that do one thing well and stay out of your way.

I care about clean, minimal, reliable code. Not clever code. Not overengineered code. Code that's easy to read in six months and doesn't surprise you in production.

Modern software frustrates me. Most of it phones home, locks you in, and treats you as a data point rather than a user. I build open alternatives — first for myself, eventually for anyone who wants software that actually belongs to them. Long-term, I intend to replace every proprietary tool I depend on with something I wrote and understand.

Open source isn't a hobby for me. It's the direction things should go.
